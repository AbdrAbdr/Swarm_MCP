Для того чтобы всё заработало, тебе нужно прописать этот сервер в конфигурации твоего MCP-клиента (например, в Claude Desktop или Cursor).

Поскольку наш сервер написан на Python с использованием библиотеки FastMCP, он запускается как локальный процесс (через stdio).

Вот как это нужно прописать и как запустить:

1. Настройка конфигурации (JSON)
Добавь этот блок в файл настроек (например, claude_desktop_config.json или настройки Cursor):

{
  "mcpServers": {
    "mcp-orchestrator": {
      "command": "python",
      "args": [
        "C:/Users/abdr/Desktop/MCP/mcp-orchestrator/server.py"
      ],
      "env": {
        "PYTHONPATH": "C:/Users/abdr/Desktop/MCP/mcp-orchestrator"
      },
      "disabled": false,
      "enabled": true
    }
  }
}
Важно: Если ты используешь Mac, путь будет другим (например, /Users/имя/Desktop/...). На Mac команда тоже будет python3.

2. Подготовка окружения (что нужно запустить один раз)
Перед первым запуском тебе нужно установить необходимые библиотеки в ту среду, где будет работать скрипт:

pip install mcp websockets
(Или через uv, если он у тебя установлен: uv pip install mcp websockets)

3. Документация (README.md)
Я создал файл документации рядом с твоим сервером. В нем расписано, как пользоваться системой.

[tool_call: write(filePath="C:\Users\abdr\Desktop\MCP\mcp-orchestrator\README.md", content="

MCP Real-Time Orchestrator (RTO)
Система для координации работы нескольких ИИ-агентов на разных компьютерах (Windows/Mac) в одном Git-репозитории.

Основные возможности
Real-Time Hub: Мгновенный обмен статусами через WebSocket (Cloudflare Workers).
Git Worktrees: Каждая задача создается в отдельной папке рядом с основным проектом.
GitHub Sync: Новые ветки автоматически пушатся в облако для синхронизации между Mac и Windows.
Интеллектуальный Координатор: Модель сама анализирует задачи, задает уточняющие вопросы и составляет план.
